Você vai criar para mim um projeto chamado **TerminalVision-MCP**.

### Objetivo
Criar um MCP Server universal que dá aos agentes de IA visão e controle humano do terminal, funcionando bem com modelos que não têm visão nativa (como DeepSeek e MiniMax M2.7).

### Regras importantes sobre visão
- O MCP deve priorizar captura em **texto** (mais rápido e eficiente).
- Quando for útil ou solicitado, deve oferecer captura como **imagem** (print scr real).
- Se o modelo atual não aceitar input de imagem diretamente, o MCP deve:
  - Listar automaticamente todos os MCPs de visão disponíveis no ambiente.
  - Escolher o melhor (ou perguntar ao usuário) para descrever a imagem capturada.
  - Não hardcode nenhum MCP específico (ex: não mencione Gemma ou MiniMax). Deve descobrir e listar todos os MCPs de descrição de imagens que estiverem configurados.

### Como o TerminalVision-MCP deve funcionar

**Captura de tela:**
- Modo primário: texto completo via PTY + emulador VT100/headless (buffer visível + cursor + hash da tela).
- Modo imagem: tira screenshot real da janela do terminal e retorna o caminho da imagem.

**Ferramentas que deve expor:**
- terminal_spawn(command, cwd, cols=120, rows=40)
- terminal_get_screen(session_id, format="text" | "image")
- terminal_send_keys(session_id, keys)  // suporta Ctrl+P, Esc, setas, sequências complexas
- terminal_wait_for_stable(session_id, timeout_ms)
- terminal_list_sessions()
- terminal_resize(session_id, cols, rows)
- terminal_kill(session_id)

**Comportamento inteligente**
- Sempre que possível, usar texto.
- Quando o agente pedir visão mais detalhada ou quando texto não for suficiente, usar imagem + chamar um MCP de descrição de imagens (descobrindo quais estão disponíveis).
- Excelente detecção de estabilidade de tela (comparar frames, evitar redraws parciais).

### Requisitos técnicos
- Funcionar com qualquer TUI (meu editor custom, Vim, htop, etc.).
- Suporte a múltiplas sessões persistentes.
- Robustez alta (redraws, alternate screen, Unicode, etc.).
- Fácil de adicionar em Qwen Code, Claude Code, etc. (incluir instruções claras no README).

Use Node.js/TypeScript ou Python (escolha a que ficar mais robusta). Crie o projeto completo com README explicando tudo, como instalar, rodar e como o sistema decide entre texto e visão.

Comece a construir seguindo exatamente essa especificação.
